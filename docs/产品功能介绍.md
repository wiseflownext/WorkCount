# 工计宝 - 制造业计时计件薪资管理APP

## 一、产品定位

面向制造业的**纯安卓客户端**薪资管理工具，支持**计件**和**计时**两种结算模式，目标用户为40-60岁制造业员工，操作极简、大字体、零学习成本。

## 二、技术方案

| 层级 | 技术选型 | 说明 |
|------|---------|------|
| 客户端 | React Native (Expo) | 安卓优先，跨平台能力保留 |
| 后端服务 | Supabase (BaaS) | PostgreSQL + Auth + Realtime，免运维 |
| 认证 | Supabase Auth | 手机号 + 密码登录 |
| 数据导出 | xlsx.js | 前端生成Excel文件 |
| 状态管理 | Zustand | 轻量状态管理 |
| UI组件 | React Native Paper | Material Design风格，适配大字体 |

## 三、角色权限

### 管理员
- 员工管理（增删改查、设置时薪）
- 产品/工序管理（增删改查、设置单价）
- 查看/管理所有员工数据
- 日/周/月报表查看
- 数据导出Excel
- 审核/修改员工提交的记录

### 普通员工
- 计件录入（选产品/工序 → 填数量 → 提交）
- 计时打卡（选工作内容 → 开始 → 结束）
- 查看自己的日报表/工资条

## 四、核心功能模块

### 4.1 员工管理
- 管理员统一录入员工（姓名、手机号、初始密码）
- 设置每个员工的时薪（元/半小时）
- 员工状态：在职/离职

### 4.2 产品/工序管理

| 字段 | 说明 |
|------|------|
| 编号 | 唯一编码，如 P001 |
| 名称 | 产品或工序名称 |
| 类型 | 产品 / 工序 |
| 计件单价 | 元/件 |
| 单位 | 件、个、组、套等 |
| 状态 | 启用 / 停用 |
| 备注 | 可选补充说明 |

**单价变更规则**：新单价仅对变更后的记录生效，历史记录保留原单价快照。

### 4.3 计件模块
- 员工从预设列表选择产品/工序
- 输入完成数量
- 系统自动计算金额 = 数量 × 当前单价
- 每条记录保存单价快照，不受后续单价变更影响
- 支持当日多次录入不同产品/工序（最多3种常用）

### 4.4 计时模块
- 员工选择/填写工作内容（可关联产品/工序）
- 点击「开始工作」按钮开始计时
- 点击「结束工作」按钮停止计时
- 时长按**半小时**为最小单位（不足半小时按半小时算）
- 金额 = 时长(半小时数) × 时薪(元/半小时)
- 工作中状态有明显视觉提示

### 4.5 互斥规则
- 同一员工同一时间段，计件和计时**不可同时进行**
- 计时进行中时，不可录入计件
- 有未结束的计时记录时，提示先结束

### 4.6 报表模块

**日报表**
- 员工视角：当日所有计件/计时记录明细 + 日薪合计
- 管理员视角：所有员工当日数据汇总，支持按员工筛选

**周报表**
- 本周每日薪资汇总 + 周合计
- 管理员可查看全员周报

**月报表**
- 本月每日薪资汇总 + 月合计
- 即工资条，包含计件总额、计时总额、总计

### 4.7 数据导出
- 支持导出为Excel文件
- 可按时间范围、员工筛选导出
- 包含明细数据和汇总数据

## 五、数据库表结构

### users（用户表）
```
id, phone, password_hash, name, role(admin/worker), 
hourly_rate(时薪), status(active/inactive), created_at
```

### products（产品工序表）
```
id, code, name, type(product/process), unit_price, 
unit(件/个/组/套), status(active/inactive), remark, created_at
```

### piece_records（计件记录表）
```
id, user_id, product_id, quantity, unit_price_snapshot, 
amount, work_date, created_at
```

### time_records（计时记录表）
```
id, user_id, product_id(nullable), work_content, 
start_time, end_time, duration_half_hours, hourly_rate_snapshot,
amount, work_date, status(working/completed), created_at
```

### price_history（单价变更历史表）
```
id, product_id, old_price, new_price, changed_by, changed_at
```

## 七、UI设计原则

1. **大字体**：正文不小于18sp，按钮文字不小于20sp
2. **大按钮**：最小触摸区域 48x48dp，核心操作按钮更大
3. **高对比度**：深色文字 + 浅色背景，状态用颜色区分
4. **扁平层级**：首页直达核心功能，最多2层页面深度
5. **即时反馈**：操作后立即显示结果，金额实时计算
6. **防误操作**：关键操作需二次确认
